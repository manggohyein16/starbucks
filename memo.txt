화면에 출력되는 특성
    1. 인라인 요소 (글자)
    2. 블록 요소 (상자)

    1. 인라인 요소 : 제약사항이 블록라인 보다 많음
    1) span태그 : 본질적으로 아무런 의미 X
                  콘텐츠 영역 구분을 위해서 사용하는 요소
                  요소가 수평으로 쌓임(왼>오)
                  글자를 의미하는 요소(글자처럼 표기, 취급)
                  > 줄바꿈 = 띄어쓰기

        포함한 콘텐츠 크기만큼 자동으로 *줄어듬* - 가로 & 세로
          > <span style="width:100px;">Hello</span> : 요소의 가로 너비를 지정하는 css 속성
            <span style="height:100px;">Hello</span> : 요소의 세로 너비를 지정하는 css 속성
                >결과< 반응없음!
                       *글자 요소는 가로 세로 사이즈를 지정할 수 없음*
          > <span style="margin:20px 20px;">Hello</span> : 요소의 외부 여백을 지정하는 css 속성
            <span style="padding:20px 20px;">Hello</span> : 요소의 내부 여백을 지정하는 css 속성
                >결과< 위아래 사용불가 좌우는 사용가능
        
        인라인 요소는 자식요소로 블록요소를 사용할 수 없음
        (ex) <span><div></div></span> -> 불가능
             <span><span></span></span> -> 가능
    
    2. 블록요소
    1) div태그 : 본질적으로 아무것도 나타내지 않는 콘텐츠 영역을 설정하는 용도
                 요소가 수직으로 쌓임(위?>아래)
        부모 요소의 크기만큼 자동으로 *늘어남* (늘어날 수 있는 만큼 최대한 늘어나려고 함)- *가로*
        포함한 콘텐츠 크기만큼 자동으로 줄어듬 - 세로
          > <div style="width:100px;">Hello</div> : 요소의 가로 너비를 지정하는 css 속성
            <div style="height:100px;">Hello</div> : 요소의 세로 너비를 지정하는 css 속성
                >결과< 반응!
          > <div style="margin:20px 20px;">Hello</div> : 요소의 외부 여백을 지정하는 css 속성
            <div style="padding:20px 20px;">Hello</div> : 요소의 내부 여백을 지정하는 css 속성
                >결과< 반응!

       블록 요소는 자식요소로 인라인요소를 사용할 수 있음
        (ex) <div><div></div></div> -> 가능
             <div><span></span></div> -> 가능


자주 사용하는 HTML요소들 가지고 실제로 CSS, JS를 통해서 어떻게 웹사이트를 만들 수 있는지,, 중요

    <블록요소> - 레이아웃을 작업할 때 많이 사용하는 요소
    1. div태그 :  특별한 의미가 없는 구분을 위한 요소 (Division) > 영역을 잡을 때
                  굉장히 많이 사용하는 태그
    2. hn태그 : 제목을 의미하는 요소(Heading) - (ex) h1, h2, h3,,,,h6 > 숫자가 작을수록 더 중요한 제목을 의미
                (어떠한 콘텐츠의 제목을 의미)
    3. p태그 : 문장을 의미하는 요소 (Paragraph)
               문장을 구분하는 용도로 사용
               (ex) 구분하기 애매한 내용이다,, > div태그 사용해도 됨
    4. ul태그 : 순서가 필요없는 목록의 집합을 의미 (Unordered List)
       + ol태그 : 순서가 필요하는 목록 > 사용빈도가 낮음
    5. li태그 : ul의 자식태그 
                목록 내 각 항목(List Item) -> 목록과 항목 구별 > 목록(List) : 각각의 항목(Item)들이 모여서 만들어진 전체
    => ul태그는 자식태그로 li태그가 1개 이상 존재해야함
       li태그는 ul태그로 무조건 감싸져있어야함
       > 하나의 세트

    <인라인요소> : 글자를 취급하는 용도 > 예외사항 있음
    1. img태그 : 이미지를 삽입하는 요소 (Image)
                 > <img src="삽입할 이미지의 경로(이미지가 어디에 존재하는지에 대한 경로)" alt="이미지가 출력되지 못했을 때 대신해서 출력될 이미지의 이름 명시"> -> alt(alternate) > 엑박떴다,,
                    => 필수속성(해당 속성과 하나의 세트) : src, alt
    2. a태그 : 다른/같은 페이지로 이동하는 하이퍼링크를 지정하는 요소 (Anchor - 닻 : (링크를)건다)
               (ex) <a href="경로">내용</a> -> 링크를 선택한 그 페이지에서 새로운 사이트가 열림
                    <a href="경로" target"_blank">내용</a> -> 브라우저가 새롭게 열리면서 새탭에 새로운 페이지가 열림
                    ~> target 속성 : 링크를 선택한 그 페이지에서 새로운 사이트가 열리기도 브라우저가 새롭게 열리면서 새탭에 새로운 페이지가 열리기도
                                     링크 URL의 표시(브라우저 탭)위치 - 링크를 걸어서 페이지를 이동할 때 그 새로운 페이지가 어디에 표시될 것인지 명시해주는 속성
                    ~> target="_blank" : 새로운 탭에 해당하는 페이지를 열겠다,, > 많이 사용하는 속성 & 값
    3. span태그 : 특별한 의미가 없늕 단순한 구분을 위한 요소 > 특정한 글자부분을 묶어낼 때
                  => 특별한 의미 없이 어떤 영역을 구분지어서 묶어낼때 사용하는 요소
    4. br태그 : 줄바꿈 요소 (break) - 인위적으로 줄바꿈 처리할 때 사용(코드 상에서 단순하게 엔터 입력하여 줄바꿈 처리 안됨)

    input태그 : 사용자가 데이터를 입력하는 요소(Inline-block)
